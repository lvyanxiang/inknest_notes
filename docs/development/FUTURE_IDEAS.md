# InkNest Notes Future Ideas

这里记录暂不纳入当前开发里程碑，但值得继续探索的产品想法。

## 产品愿景：从要做的事开始，而不是从空白页开始

市场上的笔记应用通常把“新建笔记”作为主要入口：用户先创建一张空白
笔记，再进入编辑器寻找录音、扫描、PDF 或 AI 等功能。

InkNest Notes 后续应采用不同的产品思路：

> 不从空白页开始，从用户正在做的事开始。

首页除了保留笔记库和普通的新建按钮，还应直接列出用户可以完成的事情。
用户点击某个功能后，InkNest 自动创建一份笔记，为它启用对应能力，并
直接进入匹配的工作空间。功能运行产生的录音、转写、笔迹、图片和 AI
结果都归属于这份笔记，而不是成为用完即丢的临时工具结果。

### 首页功能入口

#### 记录

- 快速手写：创建空白手写笔记并立即进入画布。
- 实时转录：创建转录笔记，进入录音与实时字幕界面；用户确认开始后，
  同时保存原始录音、分段文字和时间戳。
- 录音笔记：创建以音频为主的笔记，允许边录边写，并将笔画与播放时间
  对齐。
- 扫描成笔记：打开相机扫描纸张或白板，自动裁边、校正和 OCR，然后
  创建可继续批注的笔记。
- 图片转笔记：从相册选择图片，提取文字和结构后创建笔记。

#### 学习

- 导入 PDF 学习：选择 PDF 后自动创建学习笔记，直接进入阅读、批注、
  书签和搜索工作流。
- 课堂笔记：创建同时包含录音、实时转录、手写和课件的课堂工作空间。
- 整理课堂内容：选择一份已有课堂笔记，生成带来源的摘要、知识大纲和
  待复习内容。
- 生成闪卡：选择笔记或页面后创建复习卡片集，每张卡保留原始页码或
  录音时间。
- 生成测验：从指定笔记生成选择题、填空题和简答题，并创建可重复练习
  的测验笔记。

#### 工作

- 会议记录：创建会议笔记，启用录音、实时转录、发言人区分、会议摘要
  和待办事项提取。
- 访谈记录：创建带问题大纲、录音、逐字稿和重点标记的访谈笔记。
- 拍照归档：扫描票据、合同、通知或资料，自动识别标题、日期和分类。

#### 创作与 AI

- 笔迹美化：创建或选择手写笔记，进入保留个人风格的笔迹修整流程。
- AI 整理：选择已有笔记，将零散内容整理成结构化的新笔记，原文保持
  不变并可随时跳回。
- 对笔记提问：先选择笔记范围，再进入带来源引用的问答空间。
- 润色与翻译：创建文本笔记或选择已有内容，预览改写结果后再决定是否
  替换。
- 灵感捕捉：通过一句话、语音或图片快速创建笔记，稍后由 AI 推荐标题、
  标签和关联笔记。

### 实时转录入口示例

用户点击首页的“实时转录”后：

1. 系统立即创建一份转录笔记，标题可暂定为当前日期和时间。
2. 进入专用工作空间，显示录音控制、实时字幕和可书写区域。
3. 用户确认开始后再申请或使用麦克风，避免点击入口便意外录音。
4. 转写文字按时间分段保存，手写笔画也记录对应时间。
5. 点击任意文字或笔画，可以跳转到相关录音位置。
6. 结束录音后，可生成摘要、关键词、待办事项、闪卡或测验。
7. 所有结果继续保存在同一份普通笔记中，用户仍可编辑、移动、导出和
   搜索。

### 统一的创建规则

- 普通“新建笔记”仍然保留，但不再是唯一或最显眼的创建方式。
- 每个功能入口最终都创建标准的 `Notebook`，避免为每种场景维护完全
  不同的数据结构。
- 场景入口本质上是“笔记模板 + 默认开启的能力 + 专用首屏”。
- 用户进入场景后仍可使用通用的手写、文本、图片、PDF 和页面功能。
- 如果用户没有产生任何内容便退出，应自动清理空的临时笔记。
- AI 生成的新笔记必须保存来源笔记、页码或录音时间，形成可追溯关系。
- 首页优先展示高频场景和最近使用的功能，并允许用户自定义排序。

### 产品差异

InkNest Notes 不只是一套放在编辑器工具栏里的功能，而是一个帮助用户
直接开始任务的笔记工作台：

- 传统路径：新建笔记 -> 进入编辑器 -> 寻找功能 -> 开始记录。
- InkNest 路径：选择要做的事 -> 自动创建笔记 -> 立即开始。

这种设计减少了选择和配置步骤，也让录音转写、PDF 学习、扫描归档、
笔迹美化和 AI 整理成为产品一级入口，而不是藏在多层菜单里的附属功能。

## 1. 个性化笔迹美化

让用户能够将自己的手写笔迹变得更工整、更漂亮，同时尽量保留原有的
书写风格和个人辨识度。

理想效果包括：

- 平滑抖动或不稳定的笔画。
- 调整字形大小、间距、倾斜和基线，使整段文字更整齐。
- 优化笔画粗细与连接效果，让书写看起来更自然。
- 提供不同程度的美化强度，避免所有用户的笔迹都变成同一种字体。
- 默认保留原始笔迹，并支持预览、撤销和恢复。

这项能力不同于将笔迹识别为文字后替换成手写字体。它更强调学习用户
本人的书写特征，在保留个人风格的基础上，对原始笔迹进行智能修整。

### 推荐方案

优先评估 MyScript iink SDK 的 Handwriting Generation。它能够从用户的
手写样本估计个人书写风格，生成相似的数字笔迹，并提供笔迹美化能力，
与本功能的目标较为接近。

InkNest 是 Flutter 应用，接入时需要通过 iOS 和 Android 原生绑定建立
平台通道，并先确认 SDK 授权费用、中文效果、离线资源大小和导出笔画
格式是否满足要求，同时验证目标版本是否在 iOS 和 Android 都开放所需
的 Handwriting Generation 能力。

如果 MyScript 的效果、授权或数据格式不合适，再考虑自研方案：

1. 使用手写识别模型判断字符、词语和行的边界。
2. 用几何算法先完成抖动平滑、大小统一、间距和基线调整。
3. 收集用户主动提供的笔迹样本，训练个性化笔迹生成或笔画修整模型。
4. 通过 Core ML 或 ONNX Runtime 在设备端运行，避免持续上传原始笔迹。

Google ML Kit Digital Ink Recognition 可作为离线手写识别组件，但它的
主要职责是将笔画识别为文字，并不能单独完成个性化笔迹美化。

## 2. AI 能力规划

AI 不应作为一个笼统的“聊天按钮”接入。不同任务应使用各自擅长的
模型，并通过统一接口组合成完整的笔记工作流。

以下模型名称是截至 2026-07-02 的候选方案。实现时必须放在服务端配置
中，不应硬编码到 Flutter 客户端，以便后续更换模型或供应商。

### 功能与模型分工

| 功能 | AI 要完成的工作 | 推荐方案 |
| --- | --- | --- |
| 个性化笔迹美化 | 学习用户笔迹特征，重新生成或修整笔画 | 首选评估 MyScript iink Handwriting Generation；不满足要求时使用 Python 训练自有笔画模型，再导出为 Core ML 或 ONNX 模型 |
| 手写内容识别 | 将实时采集的笔画转换为可搜索文字 | Google ML Kit Digital Ink Recognition，默认设备端离线运行；数学公式、图表和更复杂的交互笔迹可评估 MyScript iink |
| 图片与扫描件 OCR | 提取相机扫描、图片型 PDF 中的中英文及版面位置 | Google ML Kit Text Recognition v2；普通 PDF 已有文本层时直接提取，不重复调用 AI |
| 录音转写 | 将课堂、会议录音转换为带时间戳的文字 | 日常转写使用 OpenAI `gpt-4o-mini-transcribe`；需要区分发言人时使用 `gpt-4o-transcribe-diarize` |
| 自动摘要与大纲 | 生成页面、录音、文档或整本笔记的摘要和结构 | 日常任务使用 OpenAI `gpt-5.4-mini`；简单标题、标签和分类可使用 `gpt-5.4-nano`；复杂的跨文档分析再按需使用 `gpt-5.5` |
| 语义搜索 | 找到表达不同但含义相近的笔记内容 | 保留 SQLite 全文检索处理精确关键词，并使用 OpenAI `text-embedding-3-small` 生成向量补充语义检索 |
| 笔记问答 | 根据用户选中的笔记、PDF 和录音回答问题 | 使用检索增强生成：先用关键词和向量搜索找出原文，再交给 `gpt-5.4-mini` 回答；复杂问题可升级到 `gpt-5.5` |
| 闪卡与测验 | 从笔记生成问答卡、选择题、填空题和复习重点 | `gpt-5.4-mini`，输出固定 JSON 结构；每道题必须保存来源页码或录音时间 |
| 智能整理 | 推荐标题、标签、关联笔记、知识点和双向链接 | `text-embedding-3-small` 负责相似度，`gpt-5.4-nano` 或 `gpt-5.4-mini` 负责命名、分类和解释关联原因 |
| 润色与翻译 | 改写选中文字、纠错、翻译并保留原文 | `gpt-5.4-mini`；只处理用户明确选中的内容，结果先预览再替换 |
| 图片、图表和页面理解 | 解释图表、板书、页面结构或图片中的知识点 | 使用支持图片输入的 `gpt-5.4-mini`；基础 OCR 仍交给 ML Kit，不用大模型做昂贵的逐页识字 |
| 公式与手绘图识别 | 将手写公式转换为 LaTeX，将草图转换为规范图形 | 优先评估 MyScript iink 的 Math、Diagram 和 Shape 能力；语言模型只负责后续解释，不负责精确还原几何结构 |

### 中国大陆与国际版候选

同一种能力可以根据部署地区切换供应商。产品层只依赖 Provider 接口，
不能让页面逻辑直接依赖某家模型的请求格式。

国际版候选：

- 日常文本任务：OpenAI `gpt-5.4-mini`。
- 高难度跨文档分析：OpenAI `gpt-5.5`。
- 语音转写：OpenAI `gpt-4o-mini-transcribe`；多人录音使用
  `gpt-4o-transcribe-diarize`。
- 文本向量：OpenAI `text-embedding-3-small`。

中国大陆版候选：

- 日常摘要、问答、闪卡和文档处理：阿里云百炼 `qwen3.7-plus`。
- 低成本标题、标签和分类：阿里云百炼 `qwen3.6-flash`。
- 复杂推理：`qwen3.7-max` 或 DeepSeek `deepseek-v4-pro`，应使用同一
  中文评测集比较准确率、延迟和成本后再决定。
- 录音转写：需要时间戳时评估 `qwen3-asr-flash-filetrans`；需要中文
  方言、热词或说话人分离时优先评估 `fun-asr`。
- 文本向量：阿里云百炼 `text-embedding-v4`。
- 云端图片和文档理解：`qwen-vl-ocr` 或 `qwen3.6-flash`；设备端基础
  OCR 仍优先使用本地 SDK。

向量模型一旦切换，已有笔记通常需要重新生成索引。因此索引中必须保存
供应商、模型名称和向量维度，不能将不同模型生成的向量混在同一索引中。

### 组合后的用户功能

这些基础模型可以组合成更有价值的功能：

- 课堂录音与笔迹同步：录音转写后，按时间戳关联用户当时写下的笔画。
- 一键生成课堂笔记：合并录音转写、手写识别和 PDF 内容，生成摘要与
  知识大纲。
- 对笔记提问：回答时显示所依据的笔记页、PDF 页码或录音时间，用户可
  点击跳回原文。
- 智能复习：从用户指定范围生成闪卡和测验，并根据错题安排后续复习。
- 跨笔记知识关联：发现不同笔记中相同或相关的概念，推荐双向链接。
- 笔迹美化后继续搜索：美化只改变显示笔画，识别文本单独保存并用于
  搜索、摘要和问答。

### 不需要 AI 的功能

以下功能使用确定性算法通常更快、更便宜，也更可靠：

- 原始笔画采集、渲染、撤销、保存和 PDF 导出。
- 第一阶段的曲线平滑、压感处理、直线吸附和基础形状修正。
- 录音时间与笔画时间的绑定。
- PDF 自带文本层的提取与普通关键词搜索。
- 页面排序、文件夹管理、同步冲突检测和本地备份。

AI 只能提供建议或生成候选结果，不能替代这些核心数据操作。

### 接入架构

客户端与云端职责应分开：

1. Flutter 客户端负责笔画、页面、录音、缓存和用户交互。
2. ML Kit、MyScript 或自有 Core ML 模型通过原生模块在设备端运行。
3. Python AI Gateway 保存云模型密钥，并统一封装语言模型、转写和
   Embedding 服务；Flutter 客户端不得内置任何云服务密钥。
4. Python 服务负责文本切块、向量索引、任务队列、用量限制和结果缓存。
5. 每个 AI 结果保存来源信息、模型版本和生成时间，以便重新生成和排查
   错误。

建议预先定义以下可替换接口：

- `HandwritingRecognitionProvider`
- `HandwritingBeautificationProvider`
- `DocumentOcrProvider`
- `TranscriptionProvider`
- `EmbeddingProvider`
- `LanguageModelProvider`

这样可以先接入一种服务验证产品价值，后续再根据地区、价格和效果更换
供应商，而不需要重写编辑器。

### 隐私与可靠性原则

- 默认优先设备端识别；上传笔记、笔迹或录音前必须获得用户明确同意。
- 只上传完成当前任务所需的最小范围，不默认上传整本笔记。
- 原始笔迹、原文和录音永远保留，AI 结果使用独立数据层保存。
- 笔迹替换、文字改写和内容删除都必须先预览、可撤销。
- 问答和摘要需要显示来源；没有来源时明确标记为模型推测。
- 为学生笔记提供关闭云端 AI、删除云端数据和清除向量索引的入口。
- 先用真实中文笔记建立评测集，再决定最终模型，不能只看厂商宣传。

### 建议实施顺序

1. AI 0：建立 Provider 接口、Python AI Gateway、授权提示和用量统计。
2. AI 1：配合当前 Audio And Search 里程碑，完成录音转写与时间戳。
3. AI 2：接入手写识别和图片 OCR，建立可搜索文本层。
4. AI 3：加入混合搜索、向量索引和带来源的笔记问答。
5. AI 4：在稳定的文本层之上加入摘要、闪卡、标签和关联笔记。
6. AI 5：先做 MyScript 技术验证，再决定个性化笔迹美化采用商业 SDK
   还是自研模型。
