## （一）课题来源、意义与主要内容

### 课题来源

本课题为自拟课题。课题依托个人毕业设计项目 InkNest Notes，面向移动端学习、办公和资料整理场景，设计并实现一套基于 Flutter 与 Python 的跨平台数字笔记系统。当前阶段本仓库先维护 Flutter 前端项目与毕业设计文档，Python 后端、服务端工具或智能处理模块将在后续项目补充后同步纳入任务书和开题报告。

### 课题意义

随着平板电脑、触控屏和手写笔设备的普及，学生、教师和办公用户越来越多地将课堂讲义、论文资料、会议材料和个人知识整理迁移到数字环境中。传统纸质笔记书写自然，但检索、备份、跨设备使用和资料复用不便；普通在线文档便于编辑和协作，但难以完整承载手写推导、图形标注和 PDF 阅读批注等学习场景。Goodnotes、Notability、OneNote 等产品的发展说明，手写笔记、PDF 批注、音频回放、搜索、跨端同步和知识库组织已经成为数字笔记系统的重要方向。

本课题以跨平台数字笔记系统为研究对象，前端选择 Flutter 实现移动端手写与文档交互，后续 Python 部分计划承担服务接口、数据处理、检索、识别或同步等扩展能力。课题具有一定的实践价值：一方面能够满足学习和办公场景中“快速书写、可靠保存、便捷导出”的基本需求；另一方面能够通过本地优先的数据模型、清晰的文件存储结构和可扩展的功能模块，为后续 Python 服务、跨设备同步、移动端快速捕获和 Web 知识库奠定基础。

### 主要内容

本课题主要完成以下内容：

1. 需求分析与总体设计：分析手写笔记、PDF 批注和资料整理场景，确定 InkNest Notes 的功能边界、数据模型、模块划分和 Flutter/Python 技术路线。
2. 笔记库模块：实现笔记本创建、打开、重命名、复制、归档、删除、文件夹管理、搜索、排序、最近笔记和缩略图预览。
3. 手写编辑模块：实现分页画布、触控书写、画笔、荧光笔、橡皮擦、颜色和线宽设置、撤销/重做、缩放平移、手指拖动画布和 Apple Pencil 优先的交互模式。
4. 页面与元素管理：实现多页笔记、页面新增、复制、删除、排序、插入空白页、文本框、手写风格文本、图片插入、形状工具和常用笔预设工具栏。
5. PDF 学习流程：实现 PDF 导入、PDF 页面背景渲染、PDF 页面缩略图、目录、书签、范围导出和带批注 PDF 导出。
6. Python 扩展预留：后续补充 Python 项目后，完善服务接口、数据处理、搜索索引、OCR/手写识别或同步备份等模块设计。
7. 智能与检索扩展：在后续阶段实现音频录制、笔迹时间线关联、PDF 文本搜索、手写识别和 OCR 检索等 Notability/Goodnotes 风格能力。
8. 测试与总结：编写模型、存储、导出和关键交互测试，整理毕业论文、演示材料和后续优化方向。

### 范围边界

毕业设计当前可验收主体为 Flutter 前端应用，包括笔记库、手写编辑、PDF 批注、导出、本地存储和富内容编辑等功能。Python 部分作为课题系统架构的后续扩展方向，先在任务书和开题报告中明确接口与模块预留，待 Python 项目补充后再写入具体框架、接口、部署和测试内容。同步备份、真实 OCR、真实手写识别、云端 AI 和 Web 知识库属于后续扩展方向，若未在答辩前实现，应在论文中作为展望说明。

## （二）目的、要求和主要技术指标

### 目的

本课题目标是设计并实现一套基于 Flutter 与 Python 的跨平台数字笔记系统。当前阶段先完成 Flutter 前端的手写笔记与 PDF 批注能力，使用户能够在移动设备上创建本地笔记本、进行流畅手写、导入并批注 PDF、管理多页内容、导出带批注的 PDF 文件；后续补充 Python 项目，用于服务接口、检索、识别、同步或数据处理等能力，并为音频、搜索、同步和知识库能力预留扩展空间。

### 基本要求

- 系统应以笔记库为入口，支持笔记本与文件夹的基本管理。
- 编辑器应支持分页笔记模型，而不是无限画布，以便匹配 PDF、导出和学习资料整理场景。
- 手写画布应支持低延迟书写、基础工具切换、撤销/重做和可靠保存。
- PDF 导入后应能够逐页作为背景进行批注，批注内容应保持可编辑。
- 导出的 PDF 应包含空白页、PDF 背景、手写笔迹、文本、图片和图形等内容。
- 数据应优先本地保存，存储格式应便于检查、迁移和后续同步。
- 系统应具有清晰的模块结构，便于论文阐述和后续功能扩展。

### 功能指标

- 笔记库功能：创建、打开、重命名、复制、归档、恢复、删除笔记本；创建文件夹并移动笔记本；支持搜索、排序、最近笔记和缩略图。
- 书写功能：支持画笔、荧光笔、橡皮擦、颜色、线宽、撤销/重做、平滑笔迹、局部擦除、缩放和平移。
- 页面功能：支持多页笔记、页面新增、复制、删除、排序和在 PDF 页面之间插入空白页。
- PDF 功能：支持导入 PDF、按页渲染背景、显示 PDF 缩略图、读取目录、添加书签、按全本/当前页/连续页范围导出 PDF。
- 富内容功能：支持文本框、手写风格文本、图片插入、图形绘制和常用笔预设。
- 智能辅助功能：支持 Smart Ink 初版流程，即框选粗略手写、确认文字并生成可编辑的手写风格文本；后续扩展真实手写识别、OCR 和全文搜索。
- 学习辅助功能：下一阶段实现音频录制，并探索录音播放时间线与书写笔迹关联。

### 可验收指标

- 应能在 Flutter 支持的目标平台运行，进入应用后默认显示笔记库页面。
- 应能创建至少一个笔记本，进入编辑器后完成书写、翻页、保存和重新打开恢复。
- 应能导入 PDF 并生成对应笔记页面，用户可在 PDF 页面上书写或插入内容。
- 应能导出包含页面背景、笔迹、文本、图片和图形的 PDF 文件，并支持全本、当前页或连续页范围导出。
- 应能通过自动化测试覆盖核心模型、文件仓储、笔迹几何和 PDF 导出逻辑。
- 文档应说明 Python 模块的后续职责边界；若 Python 项目在答辩前补充，应提供接口说明、运行方式和基础测试记录。

### 非功能指标

- 可用性：界面优先适配 iPad 横屏和触控使用，常用工具应容易触达，手指平移与手写输入不应冲突。
- 可靠性：笔记本元数据和页面内容以 JSON 文件保存；文件写入采用临时文件替换和页面保存串行化，降低高频编辑时的数据损坏风险。
- 性能：PDF 背景渲染应避免重复打开文档和重复渲染，导出时缓存相同背景；普通笔记页面书写、缩放和缩略图显示应保持流畅。
- 可维护性：代码按 `lib/app`、`lib/features`、`lib/models`、`lib/storage`、`lib/export` 等目录组织，使用仓储层隔离 UI 与存储实现。
- 可扩展性：数据模型预留文本、图片、图形、Smart Ink、音频、搜索、同步和 Web 知识库等后续扩展点；Python 项目补充后，应补充前后端接口、数据交换格式和服务部署说明。
- 可测试性：模型、文件仓储、笔迹几何、PDF 导出等核心模块应具有自动化测试；文档变更至少通过 `git diff --check` 检查。

### 预期成果

- 一套可运行的 Flutter 数字笔记前端应用原型。
- 一套本地笔记数据模型与文件存储方案。
- 一套 PDF 导入、批注和导出流程。
- 一份 Python 后续扩展模块设计说明或项目补充记录。
- 毕业论文、任务书、开题报告、演示材料和测试记录。
